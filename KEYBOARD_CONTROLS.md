# Управление с клавиатуры

Список всех клавиатурных и мышиных привязок, добавленных в проект, и за что каждая отвечает.

## Ручное управление роботом (`RobotBrain.Heuristic`)

Активно, когда на `Behavior Parameters` робота выставлен `Behavior Type = Heuristic Only`. Работает одинаково для симулированного робота и для реального (если на `RobotBrain` включён `Use Real Robot Io` — команда в обоих случаях идёт через один и тот же `Heuristic()`, дальше её забирает нужный `IRobotCommandSink`).

| Клавиша | Действие |
| --- | --- |
| `W` / `↑` | Газ вперёд |
| `S` / `↓` | Газ назад |
| `A` / `←` | Руль влево |
| `D` / `→` | Руль вправо |
| `G` | Схватить мяч (закрыть клешню) |
| `R` | Отпустить мяч (открыть клешню) |

## Аварийная остановка реального робота (`RobotRosTeleop`)

Компонент подключается автоматически при старте Play в сценах, где нет активного обучения (`RobotBrain.isTraining = false` или робота нет вовсе) — во время обучения он не устанавливается вообще, чтобы случайно не отправить команду на физического робота.

| Клавиша | Действие |
| --- | --- |
| `Escape` | Экстренная остановка (E-STOP) — прекращает публиковать команды движения реальному роботу |
| `Enter` | Снять экстренную остановку |

## Генерация арены (`EnvironmentManager`)

| Клавиша | Действие |
| --- | --- |
| `Z` | Немедленно перегенерировать стены/путь/препятствия **на всех аренах сразу** (обычно это происходит автоматически раз в `Regenerate Every Episodes` эпизодов) |

## Камера обзора (`TrainingCameraController`)

Управляет `Camera.main` в Game-вкладке — полезно, когда арен много (`ArenaSpawner.arenaCount`) и нужно разглядеть конкретную из них.

| Ввод | Действие |
| --- | --- |
| Колесо мыши | Зум (изменяет `orthographic size`, в границах `Min/Max Orthographic Size`) |
| Средняя кнопка мыши + перетаскивание | Свободное панорамирование вдоль пола (как в Scene-вью самого Unity) |
| `I` / `K` | Панорамирование вперёд/назад (клавиатурная альтернатива перетаскиванию) |
| `J` / `L` | Панорамирование влево/вправо |
| `]` (или `PageDown`, если есть) | Переключиться на следующую арену — камера приближается к её центру (`Focused Orthographic Size`) |
| `[` (или `PageUp`, если есть) | Переключиться на предыдущую арену |
| `Backspace` (или `Home`, если есть) | Вернуться к общему виду всей сетки арен |

## Ручное управление цифровым двойником без ML-Agents (`ManualController`)

Компонент **отключён по умолчанию** (`m_Enabled: 0`) в префабе робота — предназначен для отдельного тестирования цифрового двойника (например, во время сборки арены), не участвует в обучении. Управляет только гусеницами напрямую, без клешни.

| Клавиша | Действие |
| --- | --- |
| `W` / `↑` | Газ вперёд |
| `S` / `↓` | Газ назад |
| `A` / `←` | Руль влево |
| `D` / `→` | Руль вправо |

## Известные пересечения клавиш

- `W/A/S/D` и стрелки используются и в `RobotBrain.Heuristic()`, и в `ManualController` — конфликта на практике нет, так как `ManualController` выключен по умолчанию и оба компонента не работают на одном роботе одновременно в обычном сценарии.
- `G`/`R` в `RobotBrain.Heuristic()` не пересекаются ни с чем другим — старое отдельное чтение `G`/`R` в `GripperController` было убрано при объединении с master, теперь захват идёт только через `RobotBrain`.
